Project Overview: Pilates Studio Management System
Description

This project features a robust relational database developed using Python and SQL, designed to manage a professional Pilates studio. The system is architected to handle complex real-world scenarios, including extensive equipment inventories and diverse Pilates training methodologies.

Database Schema & Architecture
The schema comprises 10 interconnected tables that cover all essential business domains:

Core Entities: Clients, Trainers, and Sessions.

Operational Management: User Accounts, Activity Logs (Audit Trails), and Scheduling (Appointments).

Financial & Inventory: Payments, Subscriptions, and Equipment Management.

The relationships between these entities are structured to ensure data integrity and are detailed in the attached Entity-Relationship Diagram (ERD).

Technical Advanced Features
The system is currently in a phase of continuous development, featuring high-level automation and optimization:

Automation: Automated database connection handling and dynamic data population scripts designed to test container memory limits through bulk loading.

Optimization & Performance: Implementation of Stored Procedures, Triggers, and Indexes.

Benchmarking: A comprehensive performance analysis comparing query execution times with and without active indexing to validate optimization strategies.

Web Integration: A web interface developed using the Flask framework, bridging the backend database logic with a functional frontend.

Technology Stack

Database: SQL, MariaDB, Docker.

Backend: Python.

Frontend: HTML5, CSS3 (Jinja2/Flask templating).

Development Environment: PyCharm & DataGrip (JetBrains), utilized for direct SQL querying and advanced schema visualization.
